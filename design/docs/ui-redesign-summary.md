# 知期 UI 重构整理

## 现用功能梳理

### 1. 今日总览
- 周期阶段与距下次月经天数
- 易孕窗口提醒
- 今日记录摘要
- 近期周历与阶段标记
- 快速跳转到记录与洞察

### 2. 记录中心
- 月历翻页与按日查看
- 月经来了 / 月经走了 快捷登记
- 指标录入：流量、疼痛、情绪、白带、体温、体重、日记、睡眠、便便、药物
- 同房记录：时间、防护方式、备注

### 3. 洞察与健康内容
- 最近记录覆盖率
- 周期阶段解读
- 个性化提示
- 科普卡片、FAQ、风险信号

### 4. 个人与隐私
- 周期参数设置
- PIN 解锁与自动锁定
- 每日提醒与隐私通知
- 本地导入/导出备份
- 算法说明与数据状态

### 5. 桌面能力
- 周期提醒 Widget

## 现状问题
- 首页、记录、洞察的职责交叉，信息架构不够清晰。
- 底部导航语义不统一，`insights` 与 `analysis` 实际承载内容容易混淆。
- 快速记录入口分散，新增动作成本偏高。
- 我的页面功能很多，但缺少优先级分组和状态可视化。
- 视觉语言偏“统一玻璃卡片”，层级感和页面识别度不足。

## 新版信息架构
- `今日 Today`：今天的周期状态、提醒、快捷记录、今日摘要
- `记录 Journal`：按日记录、月历浏览、快捷录入、同房记录
- `趋势 Trends`：趋势、覆盖率、健康提示、风险信号
- `安全 Vault`：周期设置、提醒、解锁、备份、数据状态

## 设计方向
- 视觉基调：暖纸感 + 深酒红 + 苔绿色 + 黄铜色强调
- 版式策略：上层大标题、下层模块化卡片、减少均质玻璃感
- 交互策略：记录优先，任何主页面 1 步内可进入新增记录
- 文案策略：更像个人健康日志，而不是工具面板
